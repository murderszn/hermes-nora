Shipped

Pollinations AI — default brain

Nora's app-side reasoning runs on Pollinations AI so users need one key, not a provider maze. Activate wires custom:pollinations with kimi (256K, tools, vision). Onboard prompts for POLLINATIONS_API_KEY and syncs to Hermes.

Why Pollinations for Nora

  • Easy onboarding — single API key at enter.pollinations.ai
  • Multimodal default — kimi handles chat, tools, and images on one endpoint
  • Model menu — switch to deepseek, glm, claude-fast in chat without reconfiguring Hermes
  • Hermes-compatible — OpenAI-style API; existing users keep their provider via --merge-only
In progress

Hermes harness & personalization SDK

Nora is evolving from a persona pack into a full harness for building and shipping customized Hermes agents — soul, skills, config defaults, channel wiring, and tool packs — without forking Hermes itself.

Five phases to a complete harness

  1. Phase 1 — Persona pack (shipped): SOUL.md, skills library, Pollinations + Discord defaults, activate/onboard/doctor CLI.
  2. Phase 2 — Tool harness (in progress): curated skills for browser use, computer/terminal use, MCP, and common CLI patterns — wired to Hermes toolsets.
  3. Phase 3 — Sensory stack (planned): TTS/STT pipelines, camera and screen vision, voice gateway routes — persona-aware responses.
  4. Phase 4 — Config & interaction SDK (planned): schema-driven defaults, env templates per channel, merge-only profiles, dashboard for harness status.
  5. Phase 5 — Agent SDK (planned): fork Nora to ship your own persona; connector marketplace; web3 and domain-specific skill packs as installable modules.

Modern tool & skill stack

Hermes already ships terminal, browser, web, vision, and MCP primitives. Nora will package opinionated skills on top — evaluated against current open-source options (browser-use, Stagehand, agent-browser, Playwright MCP, etc.) with Hermes-native fallbacks.

Browser tooling research (ongoing)

We track the open-source agent-browser landscape and pick defaults that work with Hermes locally:

  • Hermes native — agent-browser, Browserbase cloud, Camofox local anti-detect
  • Agent frameworks — browser-use, Stagehand, LaVague — evaluated for skill wrappers, not runtime replacement
  • Protocol bridges — Playwright MCP, browser MCP servers — for users who already run MCP tooling

Terminal harness & config interaction

Nora is designed for terminal-first operators. The harness CLI is the interaction layer until the dashboard ships — same commands in docs, CI, and Discord “help me set up” flows.

cli

./scripts/onboard.sh

Interactive wizard — prerequisites, SOUL, Pollinations key, channel secrets, activate, gateway, doctor.

cli

./scripts/doctor.sh

Health report — wiring, secrets, gateway, optional hermes -z smoke test.

cli

./scripts/activate.sh

Merge SOUL, skills, Pollinations/Discord defaults into ~/.hermes. --merge-only for existing Hermes installs.

planned

nora config / harness TUI

Edit defaults, toggle toolsets, list MCP dirs — structured config without hand-editing YAML.

Package model

Connectors ship as installable packages with their own skills, configs, and optional dashboards. They plug into Nora's existing delegation and memory layers — no fork required. Want one we haven't listed? Open an issue or bring it to Discord.
