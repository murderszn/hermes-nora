Getting started

You do not need to be technical. You do need to install a few basics once, then follow the steps in order. Think of this as onboarding — not a computer science course.

  1. Install prerequisites

    Git, Node.js, and Python — the foundation Hermes expects. See install guides below.

  2. Install Hermes Agent

    Download the desktop app or use the terminal installer from hermes-agent.nousresearch.com. This is the engine.

  3. Install Nora

    Pull the persona addon from the home page install section or run the one-line script.

  4. Connect a channel

    Hook up Discord or Gmail first — that is how you will actually talk to her. Details in connect channels.

  5. Open the dashboard

    Launch the local control panel on your home machine. Wire in what you want Nora to manage.

  6. Say hello from anywhere

    Once a channel is live, your phone works too. Discord app or Gmail — same Nora, same memory. See your phone.

You are set up when…

  • Hermes is running on your home machine (or hub Mac mini)
  • Nora persona is loaded and the dashboard opens in your browser
  • You sent a test message via Discord or email and got a reply
  • You can reach her from your phone using that same channel

Prerequisites

Before Hermes or Nora, your machine needs a short list of tools. Most installs take under thirty minutes if you go one at a time.

Tool Why you need it Verify
Git Clone Nora, pull updates, manage local repos git --version
Node.js + npm Hermes ecosystem and many agent tools run on Node node -v && npm -v
Python 3 Skills, scripts, RPC tools, and automation Hermes delegates to python3 --version
Terminal Install scripts and occasional status checks — Terminal on Mac, PowerShell on Windows —
API key A model provider key (Nous / your chosen LLM) — Hermes walks you through this on first launch —

Hermes desktop installers bundle much of this for Mac and Windows. If you use the terminal path on Linux, install Git, Node, and Python first using the guides below.

Install guides

Copy-paste friendly steps for the basics. Pick your platform tab mentally — all three are covered in each guide.

Git

Required

Git lets you clone Nora and keep it updated. Nora's install script will fail without it.

macOS

Option A — Xcode Command Line Tools (simplest):

xcode-select --install

Option B — Download Git for Mac if you prefer a standalone installer.

Windows

Download and run the installer from git-scm.com/download/win. Accept defaults; use "Git from the command line and also from 3rd-party software."

Linux

# Debian / Ubuntu sudo apt update && sudo apt install -y git # Fedora sudo dnf install -y git

Verify

git --version

Node.js & npm

Required

Node powers large parts of the Hermes toolchain. Install the LTS version — not the bleeding-edge release.

macOS & Windows

Download the LTS installer from nodejs.org. Run it; npm ships alongside Node automatically.

Linux

# Using NodeSource (Ubuntu/Debian example) curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash - sudo apt install -y nodejs

Alternatively use your distro's package manager if it offers a recent LTS build.

Verify

node -v npm -v

Python 3

Required

Hermes delegates many tasks to Python scripts — skills, automation, data work. Python 3.10+ is a safe target.

macOS

Recent macOS versions include Python 3. If python3 --version fails, install from python.org or run:

brew install python3

Windows

Download from python.org/downloads/windows. Check "Add python.exe to PATH" during install — this is the step most people miss.

Linux

# Debian / Ubuntu sudo apt update && sudo apt install -y python3 python3-pip python3-venv # Fedora sudo dnf install -y python3 python3-pip

Verify

python3 --version pip3 --version

Hermes Agent

Core

Hermes is the agent runtime. Install it before Nora.

macOS 12+

Download the .dmg from hermes-agent.nousresearch.com, drag to Applications, and open. Follow the setup wizard for your API key and gateway.

Windows 10/11

Download the .exe installer from the same site. Run as administrator if Windows prompts you.

Linux

Use the terminal install path on the Hermes site — same pattern as Nora:

curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash

Exact URL may vary — always use the current link from the official Hermes docs.

Verify

Hermes opens, accepts your API key, and shows a connected status in the app or CLI.

Nora addon

Persona

Once Hermes is running, add Nora — persona, skills, and dashboard defaults.

macOS / Linux

curl -fsSL https://raw.githubusercontent.com/murderszn/hermes-nora/main/install.sh | bash

Windows (PowerShell)

irm https://raw.githubusercontent.com/murderszn/hermes-nora/main/install.ps1 | iex

GitHub CLI

gh repo clone murderszn/hermes-nora && cd hermes-nora && ./bootstrap.sh

Then point Hermes at the Nora directory per the bootstrap output. Full commands also live on the home page.

Verify

ls ~/.nora # or your install path # persona/, skills/, tools/ should exist after bootstrap

Your phone works too

Nora runs on your computer, not inside your phone. But once Hermes is connected to Discord or email, you talk to her through apps you already have. No special mobile app required.

The mental model

Home machine = Nora's body (always on, does the work).
Phone = your pocket remote (send messages, get replies, approve tasks).

💬

Discord

Install Discord on iOS or Android. DM your Nora bot or post in a channel she monitors. Replies land on your phone instantly.

✉️

Gmail

Give Nora her own address. Email her from your phone's mail app — great for longer tasks, attachments, and async project work.

📱

Other channels

Telegram, Slack, Signal, WhatsApp — Hermes supports a growing list. Same idea: app on phone, agent on machine.

This is how a MacBook + home hub setup feels seamless: you are on the road with your phone, Nora is on the Mac mini at home, and Discord is the bridge between you.

Connect channels

Pick one channel to start. Get it working end-to-end before you add more.

Discord (recommended first)

Chat
  1. Create a Discord server (or use an existing one) where Nora will live.
  2. In the Discord Developer Portal, create an application and bot. Copy the bot token.
  3. In Hermes settings, add the Discord gateway and paste your token. Enable the intents Hermes requests.
  4. Invite the bot to your server with message permissions.
  5. Send hello in a channel. Nora should reply.
  6. Open Discord on your phone and repeat — same conversation, same agent.

Gmail

Email
  1. Create a dedicated Gmail account for Nora (e.g. nora.yourname@gmail.com).
  2. In Google Cloud Console, create a project and enable the Gmail API.
  3. Create OAuth credentials or a service account — Hermes docs cover the exact flow for your setup.
  4. Connect the inbox in Hermes. Send a test email from your personal account.
  5. Nora replies. You can continue the thread from Gmail on your phone.

Email is slower than chat but better for project handoffs, newsletters, and anything you want an audit trail for.

Overview

Nora is Hermes Agent with a personality layer and a local control dashboard. Hermes handles memory, skills, tool use, scheduling, and multi-channel messaging. Nora adds the ops persona, dashboards, and defaults for people who want a capable assistant without living in a terminal.

Hermes Agent

The runtime. Persistent memory, skill generation, subagents, sandboxing, gateway scheduling. Built by Nous Research.

Nora persona

Voice, skills, and ops defaults as a drop-in addon. MIT licensed on GitHub.

Local dashboard

Web UI on your machine to control the agent, your computer, and anything you configure — GPUs, forums, backups, newsletters.

The local dashboard

  • Start, pause, or inspect Hermes and scheduled tasks
  • Trigger file ops, cleanup, monitoring, and backups
  • Wire custom panels: GPU jobs, Discord status, cloud sync, newsletter drafts
  • See what ran, what failed, and what is queued — without SSH

Real use cases

Patterns that work today — not slide-deck fiction.

Work agent

Two computers, one assistant

MacBook for daily work, home Mac as the hub. Discord for chat, Gmail for async. Nora has her own inbox — employee reviews become folders plus short notes; she files and organizes on the local machine.

Compute

GPU scheduling for Folding@home

RTX 5060 Ti idle most of the day. Hermes watches load — when you are not gaming, Nora spins up Folding@home for biomedical research.

Community

Discord & forum management

Nora Discord and Motion Discord today; Reddit or any forum tomorrow. Welcome flows, routine posts, escalation summaries.

Ops

Cleanup, monitoring, optimization

Disk space, stale downloads, log rotation, morning briefings — scheduled in plain language.

Backup

Cloud repos locally

Git remotes and GCP projects cloned to your machine. Nora monitors sync and reports drift.

Publishing

Saturday newsletter

You prepare content; Nora sends via Gmail API on schedule. Same pattern for digests and client updates.

Best practices for lay users

  1. Give her a real inbox. Dedicated Gmail = audit trail + calm async channel.
  2. One channel first. Discord or Gmail — not both on day one.
  3. Describe outcomes. "Make folders for each teammate and file my review notes" — not click-by-click instructions.
  4. Schedule the boring stuff. Backups, cleanup, newsletters, Folding@home — let the gateway remember.
  5. Dashboard = source of truth. If it is not wired in, Nora should not pretend it exists.
  6. Start read-only. Summaries and proposals before destructive file or send permissions.
  7. Test from your phone early. If Discord works on mobile, your setup is real.
  8. Hermes docs for tools; Nora docs for patterns. Upstream = capability. Here = how humans deploy it.

Human 1.0 → Human 2.0

Internet access once separated people by availability, not intelligence. Agents are the next layer. Human 1.0 has no always-on assistant. Human 2.0 has one reachable on Discord, accountable over Gmail, rooted on a home hub — and in their pocket via the same apps.

Nora can be your first. Local, charming, configured for your life, and awake when you are not.
